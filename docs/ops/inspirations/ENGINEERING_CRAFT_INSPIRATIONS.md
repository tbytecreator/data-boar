# Engineering craft inspirations (people, products, narrative)

**Português (Brasil):** [ENGINEERING_CRAFT_INSPIRATIONS.pt_BR.md](ENGINEERING_CRAFT_INSPIRATIONS.pt_BR.md)

Purpose: capture **maintainer-chosen** references to individuals or teams whose **craft** (code quality, security awareness, shipping real products, documentation depth, and clear public narrative) informs how we think about Data Boar — without turning admiration into policy.

**Deep analysis (themes, mimic/avoid, checklist):** [ENGINEERING_CRAFT_INSPIRATION_ANALYSIS.md](../ENGINEERING_CRAFT_INSPIRATION_ANALYSIS.md) · [short source note](ENGINEERING_CRAFT_ANALYSIS.md)

This file is **separate** from the [Security/GRC baseline](README.md) (OWASP, CISA, Security Now). Overlap is fine when someone models both; keep both lists maintainable.

## Principles

- **Inputs, not mandates.** These sources inspire trade-offs and learning; product decisions still go through tests, threat model, and rollout safety.
- **Public links only.** Prefer primary sites, repos, or official channels. Do not paste private correspondence or non-public identifiers.
- **Short entries.** One or two sentences on *why* the reference matters for Data Boar (e.g. documentation style, threat modeling habit, release discipline).
- **Refresh sparingly.** Add a row when a source repeatedly proves useful; prune if it goes stale or off-topic.

## How to use

1. Skim an entry when designing a feature, doc set, or hardening slice.
1. Extract one concrete idea (e.g. “explain the failure mode before the fix”).
1. Decide: adopt now, backlog, or skip — and link to a plan, ADR, or test.
