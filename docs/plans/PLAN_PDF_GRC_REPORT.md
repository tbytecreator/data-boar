## Context

Data Boar currently generates XLSX reports ("em verso" — tabular, column-per-finding).
A companion PDF report ("em prosa") is needed for enterprise/GRC audiences who share
reports with legal, compliance, and C-suite stakeholders.

The PDF follows the structure of a GRC audit report (similar to a penetration test report
or a LGPD readiness assessment), not a spreadsheet dump.

---

## Report structure

### Page 1-3 — Executive Summary

- **Overall score** (0-100, color-coded: green/yellow/orange/red)
- **Risk level** (Low / Medium / High / Critical) with one-line justification
- **Top 3-5 violations** with severity and recommended control
- **Compliance gap summary** (LGPD articles at risk: Art. 7, 11, 46, 48)
- **Effort saved** estimate: "equivalent to X hours of manual audit by N SRE/compliance professionals"
- **Scan metadata**: date, scope (databases/files/tables scanned), Data Boar version, config used

### Pages 4-N — Detailed Findings

- Per-finding sections: location (table.column / file path), PII category, sample (redacted),
  confidence level (High/Medium/Low), recommendation, LGPD article reference
- **Heatmap** (table × column grid, colored by PII density)
- **PII category distribution** (pie/bar: CPF vs RG vs email vs health data vs financial)

### Final pages — Priority Matrix

- **Action items** ordered by: severity × exploitability × data volume
- Three tiers: **Immediate** (< 30 days), **Short-term** (30-90 days), **Strategic** (90+ days)
- Format similar to vulnerability priority matrix in security assessments

## CLI interface

```bash
# Generate PDF from latest scan (alongside XLSX, which is always generated)
uv run python main.py --config config.yaml --scan --report --report-pdf

# Generate PDF from an existing report session (no re-scan needed)
uv run python main.py --config config.yaml --report-pdf --session <session_id>

# Specify output path
uv run python main.py --config config.yaml --scan --report --report-pdf \
    --pdf-output ./reports/audit_2026-04-06.pdf
```

## API interface

```
GET /api/v1/reports/{report_id}/pdf
  → 200 application/pdf (inline or attachment)
  → 404 if report not found
  → 402 Payment Required if tier does not include PDF (Community)
```

## Notes

- XLSX is always generated (no tier restriction) — consistent with current behavior
- PDF is Pro+ feature: Community gets a watermarked preview or a "upgrade" prompt
- The PDF format should feel like a GRC deliverable, not a technical dump
- Target audience: DPO, Legal, C-suite, auditors — NOT the SRE who ran the scan
- "Hours saved" estimate: assume 1 SRE + 1 compliance person × 40h manual audit = 80h baseline
