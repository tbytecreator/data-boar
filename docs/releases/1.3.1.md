# Release 1.3.1

**Release date:** 2026-03-03

This build release follows **1.3.0** and focuses on documentation and consistency. No change to application behaviour or APIs; all 77 tests pass with no errors or warnings.

## Changes since 1.3.0

### Documentation and usability

- **Versioning convention:** Documented the **major.minor.build** scheme and bump rules in `docs/VERSIONING.md` (EN) and `docs/VERSIONING.pt_BR.md` (PT-BR). Includes a checklist of every place to update when bumping the version (pyproject.toml, about fallback, man pages, deploy examples, README/USAGE).
- **Man page section 5:** Added `docs/lgpd_crawler.5` (view with `man 5 lgpd_crawler`) describing **config and file formats**: main config topology, regex overrides, ML/DL pattern files, learned patterns, with examples. Section 1 (`man lgpd_crawler`) remains for the command and API; section 5 is for file formats and conventions.
- **Man page install:** Install instructions now include `sudo mkdir -p` for both `man1/` and `man5/` so that copying the man pages does not fail when the target directory is missing.
- **Brazilian Portuguese (pt-BR) sync:** README.pt_BR, USAGE.pt_BR, and sensitivity-detection.pt_BR were updated to match the English docs: man pages (both sections), sensitivity features (out-of-the-box sensitive categories, optional API key), Help/Health descriptions, and links to the versioning docs.

### Version and Docker

- **Version:** Bumped to **1.3.1** (build increment) in `pyproject.toml`, `core/about.py` fallback, `docs/lgpd_crawler.1`, `docs/lgpd_crawler.5`, and `deploy/DEPLOY.md` examples.
- **Docker:** Image `fabioleitao/python3-lgpd-crawler:1.3.1` and `fabioleitao/python3-lgpd-crawler:latest` are built and pushed to Docker Hub. You can run the same version from Docker without building from source.

## How to get 1.3.1

- **From source:** `uv sync` or `pip install -e .` after pulling the tag.
- **Docker:** `docker pull fabioleitao/python3-lgpd-crawler:1.3.1` or `docker pull fabioleitao/python3-lgpd-crawler:latest`.

## Full test suite

All 76 tests pass with `uv run pytest tests/ -v -W error` (no errors or warnings).
