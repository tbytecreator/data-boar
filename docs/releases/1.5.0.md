# Release 1.5.0

**Release date:** 2026-03

This release follows **1.4.3** and adds **Data Boar branding**, **trends over up to 3 previous runs**, **heatmap embedded in the Excel report**, **favicon**, and **mascot in reports and heatmap**. Application behaviour and APIs remain backwards-compatible. The full test suite (146 tests) passes with no errors or warnings.

## Changes since 1.4.3

### Branding (Data Boar)

- **Display name:** The application is branded as **Data Boar** in the UI, reports, and documentation; the Python package name remains `python3-lgpd-crawler` for compatibility.
- **Mascot assets:** Mascot images (color and B&W, SVG and PNG) are in `api/static/mascot/`. They are used in:
- **Favicon** (browser tab): PNG and generated `favicon.ico` via `scripts/build_favicon.py`
- **Web:** About, Dashboard, Reports, Configuration, and Help pages (logo next to headings and in attribution blocks)
- **README (EN and pt-BR):** Mascot at the top
- **Excel Report info sheet:** Small mascot (48×48 px) at cell D1
- **Heatmap PNG:** Small mascot in the lower-right corner
- **Favicon:** `api/static/favicon.ico` is generated from the mascot (16×16 and 32×32). Regenerate with `uv run python scripts/build_favicon.py`.
- **Retired:** The former `api/static/logo.svg` (Concept C) was removed; branding is now the Data Boar mascot only.

### Trends – up to 3 previous runs

- **Trends sheet:** The **"Trends - Session comparison"** sheet now shows **up to 3 previous runs** (Prev run 1, 2, 3 with count and date) in addition to the current run, for a better aggregate view of the trend.
- **Note column:** The Note column remains insightful: for a single previous run the message is unchanged (improvement / new or increased / no change); when 2 or 3 previous runs exist, the note summarizes the trend (e.g. "Improvement over last 2 runs", "Increase in 2 of last 3 runs – review recommended", "No change over last 3 runs; trend stable").
- **Database:** `LocalDBManager.get_previous_sessions(session_id, limit=3)` returns up to 3 previous sessions; `get_previous_session(session_id)` still returns the single previous session for compatibility.

### Report and heatmap

- **Heatmap in Excel:** The **heatmap PNG** is now **embedded in the "Heatmap data" sheet** of the Excel report (below the data table, no overlap), so users get both the table and the visual in one download. The sheet is set to **fit to one page** (letter/A4) when printed.
- **Heatmap creation order:** The heatmap PNG is created before writing the Excel file so it can be embedded in the Heatmap data sheet; the standalone PNG file is still written to the report output directory as before.

### Documentation and sync

- **Documentation:** README (EN and pt-BR), deploy docs, topology, and plan docs were updated to reflect the current state (branding, trends, heatmap in sheet, mascot usage, favicon). EN and pt-BR are kept in sync for release and Docker sections.
- **Docker:** The image is published as **`fabioleitao/data_boar`** on Docker Hub (`fabioleitao/data_boar:latest` and `fabioleitao/data_boar:1.5.0`).

### Version and tests

- **Version:** 1.5.0 in `pyproject.toml` and `core/about.py` fallback.
- **Tests:** 146 tests pass with `-W error`. A regression test ensures the Trends sheet includes Prev run 1/2/3 columns and the Heatmap data sheet exists with the heatmap image when applicable.

## How to get 1.5.0

- **From source:** `uv sync` or `pip install -e .` after pulling the tag or branch.
- **Docker:** `docker pull fabioleitao/data_boar:1.5.0` or `docker pull fabioleitao/data_boar:latest`

## Building and pushing the branded Docker image

From the repo root, after tests pass:

```bash
uv run pytest -v -W error
docker build -t fabioleitao/data_boar:latest -t fabioleitao/data_boar:1.5.0 .
docker login
docker push fabioleitao/data_boar:latest
docker push fabioleitao/data_boar:1.5.0
```
