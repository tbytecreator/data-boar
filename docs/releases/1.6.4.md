# Release 1.6.4

**Release date:** 2026-03-20

Build release celebrating a solid **maintenance and ops** sprint: Docker **Python 3.13** base, documented **merge → bump → build → Hub** flow, **Dependabot/A1** hygiene (certifi + triage for upstream-blocked pyOpenSSL/Snowflake), and **automation** so alerts surface next to Scout in **`maintenance-check.ps1`**.

## Highlights

- **Docker / Scout alignment**
- Default image base **`python:3.13-slim`** (PR **#99**).
- **[DOCKER_IMAGE_RELEASE_ORDER.md](../ops/DOCKER_IMAGE_RELEASE_ORDER.md)** — default order and variants for Hub tags vs app version.
- **Security / dependency hygiene (A1)**
- **Certifi** floor bump and lock refresh (PR **#101**).
- **pyOpenSSL** Dependabot alerts **#9/#10**: documented **upstream blocker** (`snowflake-connector-python` / `bigdata` extra); see **[DEPENDABOT_PYOPENSSL_SNOWFLAKE.md](../ops/DEPENDABOT_PYOPENSSL_SNOWFLAKE.md)** — *working software over broken resolver* until Snowflake ships a compatible cap.
- **Docs and agent ergonomics**
- PLANS / **MAINTENANCE_FRONT_OF_WORK** updated for S4, **#98–#103**, publish order.
- **SECURITY.md** / **SECURITY.pt_BR.md** link the pyOpenSSL triage doc.
- **`maintenance-check.ps1`**: open **Dependabot security alerts** (GitHub API) + hint to triage doc (PR **#103**).
- Cursor **dependabot-hygiene** rule + **dependabot-recommendations** skill: upstream-blocked path.
- **API hardening (already on main)**
- Report/heatmap path validation (PR **#98**) — baseline for this release line.

## How to get 1.6.4

- **From source:** `uv sync` or `pip install -e .`
- **Docker (branded):** `docker pull fabioleitao/data_boar:1.6.4` (or `:latest` after publish)

## Build and push Docker image

From the repo root, on **`main`** at **1.6.4**, after tests pass:

```powershell
.\scripts\check-all.ps1
.\scripts\docker-lab-build.ps1
# smoke: see docs/DOCKER_SETUP.md §7
docker login
.\scripts\docker-hub-publish.ps1 -SkipBuild
```

## Publish this release (tag + GitHub Release)

```bash
git tag v1.6.4
git push origin v1.6.4
gh release create v1.6.4 --notes-file docs/releases/1.6.4.md --title "1.6.4"
```
