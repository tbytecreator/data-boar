## Building and pushing the Docker image

From the repo root, after tests pass:

```bash
uv run pytest -v -W error --ignore=tests/tests_qa
docker build -t fabioleitao/data_boar:latest -t fabioleitao/data_boar:1.5.4 .
docker login
docker push fabioleitao/data_boar:latest
docker push fabioleitao/data_boar:1.5.4
```

## After merging the release PR (tag, Docker, GitHub release)

1. **Create and push the version tag** (from `main` after merge):

   ```bash
   git checkout main
   git pull origin main
   git tag v1.5.4
   git push origin v1.5.4
   ```

2. **Build and push the Docker image** (see commands above).

3. **Create the GitHub release** (use the notes in this file):

   ```bash
   gh release create v1.5.4 --notes-file docs/releases/1.5.4.md --title "1.5.4"
   ```

   Or in the GitHub web UI: **Releases** → **Draft a new release** → choose tag `v1.5.4`, title `1.5.4`, and paste the contents of `docs/releases/1.5.4.md`.
