# Release 1.4.3

**Release date:** 2026-03-06

This build release follows **1.4.1** and focuses on **security hardening for credentials and configuration**, plus documentation alignment and regression tests. Application behaviour and APIs remain backwards-compatible; the full security test suite passes with no errors or warnings.

## Changes since 1.4.1

### Credential encoding and config protection

- **Database connectors:** SQL connectors now **URL-encode user and password** when building connection URLs (e.g. characters like `@`, `:`, `/`, `#`), so credentials do not break URL parsing or leak into host/path segments. This applies to drivers such as `postgresql+psycopg2` and keeps behaviour consistent with SQLAlchemy URL handling.
- **MongoDB connector:** The MongoDB connector builds URIs with **URL-encoded user and password**, ensuring special characters in credentials are parsed correctly by MongoDB drivers.
- **Config endpoint protection:** When `api.require_api_key` is true, `GET /config` now **returns 401 without a valid API key**, preventing unauthenticated users from downloading configuration files that may contain credentials or other secrets.

### Security documentation and guidance

- **Security docs:** Updated **`SECURITY.md`** and added **`docs/security.md`** / **`docs/security.pt_BR.md`** with:
- A summary of the credential-encoding changes for SQL and MongoDB connectors.
- Clarified guidance for technicians on enabling `api.require_api_key`, protecting `/config`, and running behind a reverse proxy with TLS.
- References to regression tests so that future changes to connectors and routes keep these guarantees.
- **Help and topology:** The **Help page** (`api/templates/help.html`) and topology / deploy documentation were refreshed to mention API key protection for `/config`, security headers, and where to find the detailed security docs.

### Regression tests for security

- **Credential encoding tests:** New tests in **`tests/test_security.py`** assert that:
- SQL connector URL building correctly encodes user and password, and that decoding yields the original credentials.
- MongoDB connector URIs preserve credentials via percent-encoding, with no raw `@` in userinfo beyond the encoded form.
- **Config endpoint tests:** Added tests ensuring that when `api.require_api_key` is enabled, `GET /config` without the correct key returns **401**, and with the correct `X-API-Key` header returns **200**.
- **Existing security tests:** Existing tests for SQL injection resistance, session ID validation (path traversal), and YAML `safe_load` remain green, keeping the overall security baseline intact.

### Versioning and man pages

- **Version bump:** Updated the project version to **1.4.3** (build increment) in:
- `pyproject.toml` (`version = "1.4.3"`)
- `core/about.py` fallback string for `get_about_info()`
- `docs/lgpd_crawler.1` and `docs/lgpd_crawler.5` `.TH` header lines
- `docs/PLANS_TODO.md` (current state / publish notes for the corporate compliance plan)
- **Docs index:** The documentation index (`docs/README.md` and `docs/README.pt_BR.md`) continues to point to the `releases/` folder; 1.4.3 is now the latest release file alongside 1.4.1 and earlier notes.

## How to get 1.4.3

- **From source:** `uv sync` or `pip install -e .` after pulling the tag.
- **Docker:** Use the image `fabioleitao/python3-lgpd-crawler:latest`; when the 1.4.3 tag is published, you can also pull `fabioleitao/python3-lgpd-crawler:1.4.3`.
