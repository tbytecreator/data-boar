# Release 1.4.1

**Release date:** 2026-03-05

This build release follows **1.4.0** and focuses on quality tooling, documentation alignment, and CI/workflow hygiene. Application behaviour and APIs are unchanged; all 138 tests pass with no errors or warnings.

## Changes since 1.4.0

### SonarQube and quality automation

- **SonarQube / SonarCloud:** Added **`sonar-project.properties`** at repo root (sources, exclusions, Python 3.12). CI workflow (`.github/workflows/ci.yml`) runs a **Sonar** job after tests when `SONAR_TOKEN` is set; same config is used by the SonarQube extension in Cursor/VS Code for local analysis.
- **Scripts:** **`scripts/sonar_issues.py`** fetches the current issue list from the Sonar API (machine-readable: `file:line:rule:severity:message`) so fixes and automation can stay in sync with the extension. **`docs/TESTING.md`** documents how to use extension issues to fix and prevent regressions.
- **Markdown (SonarQube rules):** **`scripts/fix_markdown_sonar.py`** applies MD007, MD009, MD012, MD029, MD032, MD036, MD047, and MD060 across project `.md` files (aligned tables, trailing spaces, blanks, list style, trailing newline). Duplicate headings (MD024) were resolved in README, USAGE, and sensitivity-detection (EN/pt-BR). **`tests/test_markdown_lint.py`** accepts MD060 aligned table style; TESTING.md updated accordingly.

### Commit / PR automation

- **commit-or-pr.ps1:** Parser fix (param default `$Body` no longer uses a here-string, avoiding “missing closing }”); branch-existence check uses `$null = ...` to satisfy PSScriptAnalyzer (no unused variable). **`scripts/do_commit.ps1`** and **`scripts/do_pr.ps1`** provide predefined title/body for local commit and PR so the automation script can be run without CLI escaping issues.

### Merge conflict prevention and CI permissions

- **report/generator.py:** Resolved merge conflict with `main` by keeping the refactored flow: `generate_report()` delegates all sheet writing to **`_write_excel_sheets()`** and helpers. Module docstring and **CONTRIBUTING.md** now explain this structure so future edits stay in sync and avoid the same conflict.
- **CONTRIBUTING.md:** New subsection **“Reducing merge conflicts”**: merge or rebase `main` into your branch before opening a PR; keep `report/generator.py` sheet logic inside `_write_excel_sheets` and helpers.
- **Workflow permissions:** **`.github/workflows/ci.yml`** and **`.github/workflows/codeql.yml`** have explicit **`permissions`** blocks (`contents: read`; CodeQL also `security-events: write`, `actions: read`) so GitHub’s “Workflow does not contain permissions” warning is addressed. The Sonar job in CI has its own `permissions: contents: read`.

### Version and Docker

- **Version:** Bumped to **1.4.1** (build increment) in `pyproject.toml`, `core/about.py` fallback, `docs/lgpd_crawler.1`, `docs/lgpd_crawler.5`, `deploy/DEPLOY.md`, and `docs/PLANS_TODO.md`.
- **Docker:** Image **`fabioleitao/python3-lgpd-crawler:1.4.1`** and **`fabioleitao/python3-lgpd-crawler:latest`** are built and pushed to Docker Hub.
