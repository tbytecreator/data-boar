## How to get 1.6.1

- **From source:** `uv sync` or `pip install -e .` after pulling the tag or branch.
- **Docker (branded):** `docker pull fabioleitao/data_boar:1.6.1` or `docker pull fabioleitao/data_boar:latest`

## Building and pushing the Docker image

From the repo root, after tests pass:

```bash
uv run pytest -v -W error --ignore=tests/tests_qa
# or: .\scripts\check-all.ps1
docker build -t fabioleitao/data_boar:latest -t fabioleitao/data_boar:1.6.1 .
docker login
docker push fabioleitao/data_boar:latest
docker push fabioleitao/data_boar:1.6.1
```

## After merging / for this release (tag, Docker, GitHub release)

1. **Create and push the version tag** (from your release branch after merge or from current branch):

   ```bash
   git tag v1.6.1
   git push origin v1.6.1
   ```

2. **Build and push the Docker image** (see commands above).

3. **Create the GitHub release** (use the notes in this file):

   ```bash
   gh release create v1.6.1 --notes-file docs/releases/1.6.1.md --title "1.6.1"
   ```

   Or in the GitHub web UI: **Releases** → **Draft a new release** → choose tag `v1.6.1`, title `1.6.1`, and paste the contents of `docs/releases/1.6.1.md`.
