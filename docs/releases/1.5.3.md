# Release 1.5.3

**Release date:** 2026-03

Build release with **Phase 1 (Foundation) security hardening**: tenant/technician validation, request body size limit, and logging audit so API keys, passwords, and connection strings are never written to logs. Documentation and app capabilities are aligned with current behaviour. All 155 tests pass with no errors or warnings.

## Highlights

- **Security hardening (Phase 1):**
- **Tenant/technician validation:** Optional values from API and CLI are sanitized (max length 255, control characters removed) before storage and display in reports, reducing abuse and injection risk.
- **Request body size limit:** The API rejects requests with `Content-Length` over **1 MB** (POST `/config`, `/scan`, `/scan_database`) with **HTTP 413** to mitigate DoS via huge payloads.
- **Logging policy:** Failure details and exception messages are redacted before being written to the audit log; connection URLs and `password=` / `api_key=`-style values are masked. Scanners no longer log connection strings.
- **Documentation:** SECURITY.md updated with request body limit and logging policy; new security tests for validation, body size, and redaction. Glossary (EN + pt-BR) added for terms used in docs and config.
- **No breaking changes:** All behaviour is backward compatible; existing configs and deployments continue to work.

## Changes since 1.5.2

- **Version:** 1.5.3 in `pyproject.toml`, `core/about.py` fallback, man pages, and docs (README, DEPLOY, TECH_GUIDE).
- **Security:** `core/validation.py` — `sanitize_tenant_technician()`, `redact_secrets_for_log()`; `api/routes.py` — request body size middleware (1 MB); `core/database.py` — redact failure details before logging; `scanners/db_scanner.py`, `scanners/db_connector.py` — do not log connection URLs.
- **Tests:** New tests in `tests/test_security.py` for tenant/technician sanitization, 413 on oversized body, and secret redaction in logs. Full suite: **155 tests**, no warnings.
- **Docs:** SECURITY.md (body limit, logging policy); [docs/GLOSSARY.md](GLOSSARY.md) and [GLOSSARY.pt_BR.md](GLOSSARY.pt_BR.md) for terms (norm_tag, finding, data soup, etc.); README pitch already reflected compliance samples and tuning offer.
