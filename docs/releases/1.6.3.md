# Release 1.6.3

**Release date:** 2026-03

Build release focused on operational hardening and publish hygiene after the 1.6.2 baseline: Docker/Scout hardening, safe container host binding (`API_HOST=0.0.0.0` in image while CLI default remains `127.0.0.1`), plans/KPI automation helpers, docs/version sync, and maintenance updates.

## Highlights

- **Docker hardening and Scout-oriented fixes**
  - Base image pinned to `python:3.12.13-slim` in both build/runtime stages.
  - Explicit `wheel` upgrade verification added in image build to reduce CVE drift and make scanner evidence explicit.
- **Host-binding safety preserved**
  - Container image defaults to `API_HOST=0.0.0.0` for mapped-port accessibility.
  - Local CLI/default behavior remains `127.0.0.1` for safer local operation.
  - `core/host_resolution.py` + tests updated to cover environment-variable override flow.
- **Ops helpers and planning automation**
  - Added `scripts/plans-stats.py` (dashboard count refresh/check for `PLANS_TODO.md`).
  - CI now validates plans dashboard drift (`--check`).
  - Added `scripts/kpi-export.py` for lightweight W-KPI snapshot export from GitHub data.
- **Maintenance and consistency**
  - Ruff style synchronization pass applied on merged files.
  - `tools/license-studio` JWT dependency bumped (`github.com/golang-jwt/jwt/v5` to `v5.2.2`).
  - Release/version references aligned to 1.6.3 across package metadata, fallback, docs, and man pages.

## Changes since 1.6.2

- **Runtime/deploy**
  - Dockerfile hardening and image host behavior improvements (`API_HOST` in container).
  - Docs updated to reflect expected host behavior for CLI vs Docker image.
- **Detection/report quality follow-ups**
  - Small decoupling micro-slice in report generation/detector helpers to reduce coupling without behavior regression.
- **Planning and CI**
  - Auto-generated plans status dashboard flow integrated in local checks + CI.
  - W-KPI baseline docs and export helper added.
- **Documentation and release hygiene**
  - EN/PT-BR docs kept aligned for current version and Docker tag examples.

## How to get 1.6.3

- **From source:** `uv sync` or `pip install -e .`
- **Docker (branded):** `docker pull fabioleitao/data_boar:1.6.3` (or `:latest`)

## Build and push Docker image

From the repo root, after tests pass:

```bash
uv run pytest -v -W error --ignore=tests/tests_qa
# or: .\scripts\check-all.ps1
docker build -t fabioleitao/data_boar:latest -t fabioleitao/data_boar:1.6.3 .
docker login
docker push fabioleitao/data_boar:latest
docker push fabioleitao/data_boar:1.6.3
```

## Publish this release (tag + GitHub Release)

```bash
git tag v1.6.3
git push origin v1.6.3
gh release create v1.6.3 --notes-file docs/releases/1.6.3.md --title "1.6.3"
```

## Full validation gate

```bash
.\scripts\check-all.ps1
# or: uv run pytest -v -W error
```
