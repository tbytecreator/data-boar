# Release 1.6.2

**Release date:** 2026-03

Build release: version alignment across packaging, man pages, and deploy examples; optional **fuzzy column name** matching (FN reduction slice 3) behind config + `[detection-fuzzy]` extra; continued detection and ops documentation updates.

## Highlights

- **Optional fuzzy column match:** When `sensitivity_detection.fuzzy_column_match: true` and **`rapidfuzz`** is installed (`uv sync --group dev`, `pip install .[detection-fuzzy]`, or extra `detection-fuzzy`), the detector can promote **LOW** borderline ML/DL scores to **MEDIUM** with `pattern_detected` **`FUZZY_COLUMN_MATCH`** if the column name is similar to a sensitive training term (default confidence band below the MEDIUM threshold). **Default off** — no behaviour change without the flag and library.

- **Version and release:** 1.6.2 in `pyproject.toml`, `core/about.py` fallback, man pages (`docs/data_boar.1`, `docs/data_boar.5`), README / README.pt_BR, and Docker tag examples in DEPLOY, TECH_GUIDE, DOCKER_SETUP (EN + pt-BR).

## Changes since 1.6.1

- **Detection:** FN reduction plan priority 3 — fuzzy column vs sensitive term list (ML + DL sensitive labels), `core/fuzzy_column_match.py`, wiring in `SensitivityDetector` and `AuditEngine` / `config/loader.py`.
- **Dependencies:** Optional extra `detection-fuzzy`; `rapidfuzz` added to **dev** dependency group for CI/tests.
- **Docs:** `docs/SENSITIVITY_DETECTION.md` (+ pt-BR), `PLANS_TODO` / FN plan table; aggregation map for `FUZZY_COLUMN_MATCH`.

## How to get 1.6.2

- **From source:** `uv sync` or `pip install -e .` after pulling the tag or branch.
- **Docker (branded):** `docker pull fabioleitao/data_boar:1.6.2` or `docker pull fabioleitao/data_boar:latest`

## Building and pushing the Docker image

From the repo root, after tests pass:

```bash
uv run pytest -v -W error --ignore=tests/tests_qa
# or: .\scripts\check-all.ps1
docker build -t fabioleitao/data_boar:latest -t fabioleitao/data_boar:1.6.2 .
docker login
docker push fabioleitao/data_boar:latest
docker push fabioleitao/data_boar:1.6.2
```

## After merging / for this release (tag, Docker, GitHub release)

1. **Create and push the version tag** (from your release branch after merge or from current branch):

   ```bash
   git tag v1.6.2
   git push origin v1.6.2
   ```

1. **Build and push the Docker image** (see commands above).

1. **Create the GitHub release** (use the notes in this file):

   ```bash
   gh release create v1.6.2 --notes-file docs/releases/1.6.2.md --title "1.6.2"
   ```

   Or in the GitHub web UI: **Releases** → **Draft a new release** → choose tag `v1.6.2`, title `1.6.2`, and paste the contents of `docs/releases/1.6.2.md`.
